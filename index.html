ProjectEHR User Guide

ProjectEHR is a powerful simulation electronic medical records platform for medical students to use during clinical practice, clinical simulations, and problem-based learning sessions. This use guide is intended for both students accessing cases interactively, as well as facilitators building such cases.

1.1. Login

To begin with, open the ProjectEHR URL in Google Chrome.

(Other browsers are not officially supported at this time.)

Once you've entered your username and password, please select a department.

Your system context can be changed later.

Once you've logged in, you will be redirected to the default tab, the Patient Schedule tab.

1.2. System Tools

In the system menu at the top left, there are several system tools you can use to manage ProjectEHR:

  • UpToDate: students can use this menu item to launch a web browser that opens UpToDate.
  • Patient Lookup
  • PDMP Manager
  • Manage Departments
  • Flowsheet Definitions
  • Database Management
  • Manage Conditionals

1.3. Patient Schedule

Once you've scheduled patients for appointments, you'll be able to navigate between the different department schedules and different dates using the toolbar at the top of the schedule list.

Double-clicking on an appointment in the schedule list will open that encounter.

1.4. Patient Lookup & Create

This system tool allows you to search for patient charts or create a new chart if one does not exist in the system yet.

Navigating to the first tab in this tool window will show you several text entry fields. Filling in these fields and pressing "Find Patient" or "Accept" will open the patient chart if a matching patient is found.

To create a new patient, press the "New" button at the bottom of the tool window.

If you do not enter in any of the fields (such as name, sex, and birthdate), a Doe patient will be created with a randomized last name and a birthdate of 1/1/1890.

Once the patient chart has been created, it will be opened for you automatically in Chart Review mode.

2.1. Patient Chart

If you open a patient chart in Chart Review mode, or you've just created a new patient using the Patient Lookup system tool, you will see the main patient chart window. Several tabs are available in the main pane and the sidebar, as well as an overflow menu at the top right of the main pane.

  • Snapshot
  • Chart Review
  • Visit Info
  • Problem List
  • History
  • Results Review
  • Immunizations
  • Allergies
  • Demographics
  • PDMP
  • Flowsheet
  • Appointment Desk
  • Event Log
  • Medication Administration Record (MAR)
  • Notes
  • Create Encounter

2.2. Demographics

The demographics tab allows you to see and modify basic information about the patient, such as their name, birthdate, sex, and more. If you've just created a new patient using the Patient Lookup system tool, you may need to modify their basic information.

To begin, press the edit pencil icon at the top left, next to "Basics". Then, make the changes you need.

Once you've made changes you want to save, you MUST press the Save button at the bottom of the section.

2.3. Care Team

The care team editor can be accessed by pressing the edit icon next to "Care Team" in the patient sidebar.

You'll be able to add any provider from any department to the patient's care team.

You can select from the roles provided, or enter your own role, such as "PCP".

Once you've made changes to the care team, press "Save Changes". It's important to note that the Care Team is NOT unique to the current encounter, and will be visible across ALL encounters for this patient.

2.4. Create Encounter

To create a new encounter for this patient, open the Create Encounter tab. A new tool window will pop up.

You'll be able to set the start and end date and time for the encounter here.

Each encounter must have an associated department, provider, specialty, and type. Once you've made your selections, press "Create".

Once the new encounter is created, it will open as a new tab, separate from the Chart Review tab that was already open for this patient.

3.1. Visit Info

To edit information about the current encounter, such as reason for visit (aka "chief complaint"), open the Visit Info tab.

You can enter any number of reasons for the current visit, as well as comments to provide more details. This information will be visible in the patient sidebar when this encounter is viewed.

3.2. Appointment Desk

To create a new appointment, open the Appointment Desk tab. Here you will see a list of all existing appointments, across ALL encounters for this patient.

To create a new appointment, select the encounter you wish to link, then select the department, provider, appointment type and time, as well as location. These fields will likely be the same as what you entered when creating the encounter.

To schedule this appointment, press the "Schedule" button. Once the appointment has been created, you can find it in the Patient Schedule tab.

3.3. Notes

The Notes activity allows you to view all notes for the current encounter ONLY.

(To view ALL notes, please use the Chart Review tab.)

To create a new note for the current encounter, press the "New Note" button at the top right.

This will open the Note Editor in the sidebar, and the a block editor on the main pane for Review of Systems and Physical Exam. You can put any content you would use in a Microsoft Word document into the Note Editor, including images, tables, and links.

Once your note is complete, ensure it has a service date, note type, and service department, then select "Sign" at the bottom toolbar and press "Accept". This will save the note to the current encounter.

3.4. Problem List

The problem list tab allows you to manage the list of problems for the patient for the current encounter ONLY. To start, search for an item or press the "Add" button to bring up the Diagnosis Search tool window.

You can search for multiple diagnoses at one time by using the "Select and Stay" option. Once you are ready to proceed, press the "Accept" button.

Once your problem list item has been created, you are able to change the diagnosis code and display label independently. (For example, you might want to have "Primary osteoarthritis, unspecified site (M19.91)" display as "Osteoarthritis".)

By selecting the "Encounter Diagnosis" checkbox, this problem will become a visit diagnosis. You can also add this problem to the patient's Medical History by pressing the "Add to History" button. These changes will only take place once the "Accept" button is pressed.

3.5. Medical History

The History tab will automatically open to its first sub-tab, Medical History. Here, you will be able to view medical history items for the patient in the current encounter, and add new history items.

To add a new medical history item, press the "Add History" button. Then, press the search icon next to the Diagnosis field to open the Diagnosis Picker.

Once your medical history item has been created, you are able to change the diagnosis code and display label independently. (For example, you might want to have "Primary osteoarthritis, unspecified site (M19.91)" display as "Osteoarthritis".)

You can also add this medical history item to the patient's Problem List by pressing the "Add to Problem List" button. These changes will only take place once the "Accept" button is pressed.

3.6. Family History

The family history tab allows you to view and manage the patient's family history for this current encounter ONLY. You may add a new family member by first selecting their relationship to the patient.

To add a new problem to the family history grid, press the green "Add Problem" button. A new column will appear with the problem you entered.

To add a new family history item, press the red checkmark button in the row for the family member whose problem this is, and in the column for the problem itself.

You may also view and change comments for family members by pressing the gray notepad icon in the "Comments" column.

3.7. Social History

There are multiple social history tabs in the History tab, including:

  • Substance & Sexual
  • E-Cigarette/Vaping
  • Socioeconomic
  • Activities of Daily Living (ADL)
  • Social Documentation
  • Birth Tracking
  • Pap Tracking
  • Ob/Gyn History (for patients whose Sex is Female)

In the social documentation tab, you will be able to enter comments about the patient's social history that might not fit in any of the other tabs. You can put any content you would use in a Microsoft Word document into this field, including images, tables, and links.

3.8. Home Medications

The Home Medications tab allows you to view and manage the patient's current home medication regimen during this encounter ONLY. To begin, search for a new medication, or press the Add button.

This will pop up the Orders Picker. Please note that when the Orders Picker is opened from Medications, you will not be able to view order sets or procedure orders. You can select multiple items by pressing "Select and Stay", or add items one by one using the "Accept" button.

Once you've selected a medication order, you will be able to further adjust the dosage, route, frequency, and date range. Press "Save" to confirm your changes.

3.9. Allergies

The allergies tab allows you to view and manage all allergies for the patient in the current encounter ONLY. To begin, press the "Add Allergy" button at the top right.

Select the allergen from the drop down list (the allergen type field will auto-populate to match the allergen selected).

You may add multiple reactions, select a severity, and a reaction type. You may also add comments to add more detail. Once you've made your selection, press the "Accept" button.

Upon adding a new allergen, it will appear in the Allergies tab, as well as the patient sidebar. If the reaction severity is "High" it will display in both of these places with a yellow alert background.

3.10. Immunizations

The immunizations tab allows you to view and manage all immunizations for the patient in the current encounter ONLY. To begin, press the "Add Immunization" button at the top right.

Select the immunization from the vaccine drop-down list, and add the date received. Most other fields are optional and do not need to be entered to save a new immunization entry. Once you've made your selection, press the "Accept" button.

3.11. Flowsheets

The flowsheets tab allows you to view and manage flowsheet entries for the Flowsheet Definitions available in the system. First, select a flowsheet template from the menu bar at the top (the default is "Vitals"). The right-most column shows the "Last Filled" data (which cannot be directly edited).

To begin editing a flowsheet, find the "Now" column that appears to the right of all other previously entered columns. Select a cell and begin typing to enter a value. The cell may require you to enter a number, free text, or make a selection from a drop-down list of options.

If you make changes to the Vitals flowsheet, you'll also see them in the patient sidebar. You can also hover over the vitals section in the sidebar to see prior entries.

3.12. LLM Chat

The Chat tab allows students to initiate an AI-based virtual standardized patient (AI SP) interaction. This section is password protected, so you must first enter the password provided to you.

The Chat tab looks and functions similiarly to popular videoconferencing software (i.e. Zoom, Google Meet, MS Teams).

To begin a conversation with the AI SP, press the green call button, which will then turn red to indicate that you can hang up by pressing the same button.

The mute/unmute button also allows you to select your device's microphone and speakers, and will show a waveform icon when your voice is being picked up by the system.

To configure the patient and the current encounter for AI SP use, you must press the "Settings" menu item from the menu button next to the green call button. This tool window is also password protected, with a different password than the Chat tab itself.

Once you've entered the Model Configuration tool window, you will be able to set the voice and custom prompt for the patient for the current encounter ONLY.

The system prompt is also visible in this tool window, and can be edited, but CHANGES WILL NOT BE SAVED. You can review the full script provided to the AI SP in the right pane of the tool window.

When selecting a voice for the AI SP, helpful voice hints are provided in the drop-down list. If you'd like to listen to a generated voice sample, press the play icon next to the voice name.

3.13. Chart Review

The Chart Review tab lists ALL encounters, notes, labs, imaging, and orders for the patient, across ALL encounters, including the current encounter. In the "Encounters" sub-tab, you'll be able to select an encounter and press the "Open Encounter" button to open a different encounter in a new system tab.

Cardiac lab and imaging results will appear in the Cardiac tab, if any exist. Some other labs and imaging results may appear in the Specialty Tests tab, if any exist. You can double click a lab or imaging result to open it and view it in more detail. You can also view labs and imaging in a grid or chart using the Results Review tab.

If you open a lab result, you will see a list of components with their values and ranges. Abnormal components will be marked with a yellow background and red arrow. If a lab result has any abnormal components, the entire result is marked as abnormal, with a left-sided yellow strip. A lab result may also have narrative, impression, and comments sections.

If you open an imaging result, you will see a blue "View Images" in addition to the narrative, impression, and comments sections. Clicking this button will open the Imaging Viewer for the current imaging item.

3.14. Results Review

The results review tab allows you to view all lab components and imaging results, as a grid, for the current encounter ONLY. You can hover over a cell to view the result component with its reference range and result date. Imaging results will not have a value, and instead, will show a "picture" icon. Pressing the "New Result" button at the top right of the tab will open the Edit Results tab.

3.15. Imaging Viewer

The imaging viewer tab allows you to view medical imaging, including DICOM, PNG/JPG files, and more. This tab may pop up for radiological scans, pathology slides, and EKG strips. Right clicking the currently visible image will pop up the tool box, from where you can select a tool, such as window level, pan/zoom, and more.

3.13. Edit Results

The Edit Results tab allows you to create a new lab or imaging result for the current encounter. All labs and imaging require a date/time, test name, status, and ordering provider. In addition, they may have one or more components (each with a value, low and high range, and comment). Results can also have attached imaging (such as XR, CT, or EKG).

To begin, please search for the appropriate test or exam name (this list includes all lab and radiology orders, as well as miscellaneous orders). If your selected test or exam has components, they will also appear in the lab components table below.

For each component, you can enter value, units, low and high range, and a comment. You can also delete the component if it will not be part of this result. If you change the test or exam, it will not delete components that you have already filled in.

When uploading a non-DICOM image file (i.e. PNG, JPG, etc.), you will see a preview of the image. Once you have made your selections for this new lab or imaging result, press the "Accept" button to save it.

3.17. Orders (Sidebar)

The orders sidebar tab allows you to place new orders for the current encounter. To begin, search for an order, or press the "New" button.

If any elements of the current encounter conditionally require the "__BICEP__" order to be revealed, the green "Advance Case" button will appear at the bottom right. Pressing this button automatically places and signs the "__BICEP__" order, which will then advance the patient case.

The Browse tab of the Orders Picker allows you to select from some common orders (i.e. ED orders). The preference list and facility list tabs allow you to search the broader database of orders.

Once you find an order you're looking for, you can press "Select and Stay" to queue multiple orders, or press "Accept" to proceed with editing the single order you've chosen.

Once you add new orders to the Orders tab, you are able to edit them or delete them before signing them or pending your work. If any elements of the current encounter are conditionally revealed based on the orders you've signed, those parts of the chart will instantly become visible (i.e. in the patient sidebar, or in the Chart Review tab).

4.1. Manage Conditionals

The Manage Conditionals system tool is a powerful way to build cases by obscuring elements of the current encounter until certain orders have been placed. On the left pane, you will see a tree of all elements in the current encounter, such as Problem List, Medical History, Notes, and more.

Selecting an item will show you more details about the item in the right pane. Here, you can search for and add an order (by name OR id).

You can also require multiple orders to be fulfilled before the selected item is revealed in the patient chart (i.e. one CT brain order for a pre-intervention imaging result, and two CT brain orders for the post-intervention imaging result).

You can reorder the list of conditional orders for the selected item, and delete orders that will no longer be required to be fulfilled before the selected item becomes visible in the patient chart. If no order was placed to fulfill the conditional order requirement, you will see an empty bubble to the left of the order name.

If an order has been placed for the current encounter, and it fulfills your conditional order requirement, you will instead see a green checkmark to the left of the order name.

4.2. Database Management

The Database Management system tool allows a system administrator to insepct the entire database structure (in the left pane) and perform actions on it, such as import, export, and delete (in the right pane). You are able to export or delete the entire database.

When selecting an individual patient or encounter, you will be able to delete the selected item, or export the selected item as a file. When exporting the selected item, all linked items will also be exported, such as the department an encounter occurred in, the patient's care team providers, any appointments, and so on.

When importing a file, all records in the file will be imported. If any records have the same ID (such as a provider with ID "23" having the name "Joe" in the database, but "Andy" in the imported file), these records will be overwritten with the new values from the imported file.
